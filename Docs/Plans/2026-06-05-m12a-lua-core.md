# M12a — Lua Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A **Run Lua Script** action backed by MoonSharp, with a bidirectional `vars` bridge to the bot's variables, plus `json` and `log` — the pure (no external I/O) core of the Lua escape hatch.

**Architecture:** MoonSharp (pure-C# Lua 5.2) runs the user's script in-process. A `LuaScriptHost` builds a sandboxed `Script`, seeds a global `vars` table from `ExecutionContext.Variables`, registers `json` (parse/encode) + `log`, runs the script string, writes `vars` back, and maps a Lua error → `onFailure` (else `onSuccess`). I/O host modules (`http`/`fs`/`process`) are M12b.

**Tech Stack:** C# / .NET 10, AdbCore, `MoonSharp` 2.0.0 NuGet, xUnit.

**Reference spec:** `Docs/Specs/2026-06-05-m12-lua-scripting-design.md` (§2–4, §7, §8 M12a). **M12b (http/fs/process) is a separate later plan.**

**Merge handling:** M12a is AdbCore-only with deterministic unit tests (MoonSharp is pure managed code, so tests run real Lua) and no live deps → built compile-clean + unit-green and **self-merged** via `gh` (backend-only-slice rule).

**Adaptive note:** MoonSharp 2.0.0's API (`Script`, `DynValue`, `Table`, `CoreModules` presets, `Script.DoString`, exception types `SyntaxErrorException`/`ScriptRuntimeException`, and the built-in `MoonSharp.Interpreter.Serialization.Json.JsonTableConverter`) should be matched against the installed package. using System.Text.Json;
```csharp
using MoonSharp.Interpreter;

namespace AdbCore.Scripting;

/// <summary>Converts the bot variable system's CLR values (string/double/bool, plus JsonElement from a
/// loaded .bot) to MoonSharp <see cref="DynValue"/> and back to the canonical string/double/bool.</summary>
public static class LuaValues
{
    public static DynValue ToDynValue(object? value) => value switch
    {
        null => DynValue.Nil,
        string s => DynValue.NewString(s),
        bool b => DynValue.NewBoolean(b),
        double d => DynValue.NewNumber(d),
        float f => DynValue.NewNumber(f),
        int i => DynValue.NewNumber(i),
        long l => DynValue.NewNumber(l),
        JsonElement { ValueKind: JsonValueKind.Number } je => DynValue.NewNumber(je.GetDouble()),
        JsonElement { ValueKind: JsonValueKind.True } => DynValue.True,
        JsonElement { ValueKind: JsonValueKind.False } => DynValue.False,
        JsonElement je => DynValue.NewString(je.ValueKind == JsonValueKind.String ? je.GetString() ?? "" : je.ToString()),
        _ => DynValue.NewString(value.ToString() ?? string.Empty),
    };

    /// <summary>Maps a DynValue back to a canonical CLR value (string/double/bool), or null for nil.
    /// Tables/functions are flattened to their string form (the variable system holds scalars).</summary>
    public static object? ToClr(DynValue value) => value.Type switch
    {
        DataType.Nil or DataType.Void => null,
        DataType.String => value.String,
        DataType.Number => value.Number,
        DataType.Boolean => value.Boolean,
        _ => value.ToPrintString(),
    };
}
```

```csharp
using MoonSharp.Interpreter;
using MoonSharp.Interpreter.Serialization.Json;

namespace AdbCore.Scripting;

/// <summary>Runs a user Lua script (MoonSharp) with a bidirectional `vars` bridge to the bot variables,
/// plus `json` and `log`. Pure core — no external I/O (http/fs/process are added by M12b). The host is
/// built per Run so scripts don't share state.</summary>
public sealed class LuaScriptHost
{
    private readonly Action<string> _log;

    public LuaScriptHost(Action<string> log)
    {
        ArgumentNullException.ThrowIfNull(log);
        _log = log;
    }

    public readonly record struct Result(bool Success, string? Error);

    /// <summary>Runs <paramref name="scriptText"/>. On success, writes the final `vars` table back into
    /// <paramref name="variables"/>. A Lua syntax/runtime error (or `error(...)`) yields a failed Result.</summary>
    public Result Run(string scriptText, IDictionary<string, object> variables, CancellationToken ct)
    {
        // SoftSandbox: language features but no raw io/os/loadfile (host API is provided explicitly).
        var script = new Script(CoreModules.Preset_SoftSandbox);

        // `vars` table seeded from the bot variables.
        var vars = new Table(script);
        foreach (var kv in variables)
        {
            vars[kv.Key] = LuaValues.ToDynValue(kv.Value);
        }
        script.Globals["vars"] = vars;

        // `json` table (MoonSharp's built-in JSON <-> Table converter).
        var json = new Table(script);
        json["parse"] = (Func<string, DynValue>)(s => JsonTableConverter.JsonToTable(s, script));
        json["encode"] = (Func<DynValue, string>)(v => JsonTableConverter.TableToJson(v.Table));
        script.Globals["json"] = json;

        // `log(msg)`.
        script.Globals["log"] = (Action<DynValue>)(v => _log(v.IsNil() ? "" : v.ToPrintString()));

        // Cooperative cancellation: abort the running script when the token fires.
        using var reg = ct.Register(() => script.Globals.Get("__never__")); // placeholder; see note
        try
        {
            script.DoString(scriptText);
        }
        catch (InterpreterException ex)
        {
            return new Result(false, ex.DecoratedMessage ?? ex.Message);
        }

        // Write the final `vars` table back to the bot variables.
        foreach (var pair in vars.Pairs)
        {
            if (pair.Key.Type == DataType.String)
            {
                variables[pair.Key.String] = LuaValues.ToClr(pair.Value)!;
            }
        }
        return new Result(true, null);
    }
}
```

```csharp
using AdbCore.Execution;
using AdbCore.Scripting;

namespace AdbCore.Actions.BuiltIn;

/// <summary>Runs a user Lua script (MoonSharp) with two-way access to the bot variables via `vars`, plus
/// `json` and `log`. Completes -> onSuccess; a Lua error (or error(...)) -> onFailure with the message.</summary>
public sealed class RunLuaScriptAction : IActionDefinition, IActionExecutor
{
    public const string ScriptKey = "script";
    public const string SuccessPort = "onSuccess";
    public const string FailurePort = "onFailure";

    public string TypeKey => "scripting.runLua";
    public string DisplayName => "Run Lua Script";
    public string Category => "Scripting";
    public string Description => "Runs a Lua script with access to the bot variables (vars), json, and log.";
    public List<PortDefinition> InputPorts { get; } = new() { new PortDefinition { Name = "in", Label = "In" } };
    public List<PortDefinition> OutputPorts { get; } = new()
    {
        new PortDefinition { Name = SuccessPort, Label = "On Success" },
        new PortDefinition { Name = FailurePort, Label = "On Failure" },
    };
    public List<ConfigField> ConfigFields { get; } = new()
    {
        new ConfigField { Key = ScriptKey, Label = "Lua Script", Type = ConfigFieldType.MultilineString, DefaultValue = "" },
    };
    public bool SupportsRetry => true;

    public Task<ActionResult> ExecuteAsync(ActionExecutionContext context, CancellationToken ct)
    {
        ct.ThrowIfCancellationRequested();
        var scriptText = ConfigValues.GetString(context.Action.Config, ScriptKey);

        var host = new LuaScriptHost(context.Log);
        var result = host.Run(scriptText, context.Context.Variables, ct);

        return Task.FromResult(result.Success
            ? ActionResult.Ok(SuccessPort)
            : ActionResult.Fail($"Run Lua Script: {result.Error}"));
    }
}
```
NOTE: confirm the action's log sink is reachable as `context.Log` (the `Action<string>` passed into `ActionExecutionContext`). If the property name differs (e.g. the executor receives the log differently — check how `LogAction` emits), use that. The test `Log_EmitsToSink` is the spec.
